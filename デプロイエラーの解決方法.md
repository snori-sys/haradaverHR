# デプロイエラーの解決方法

## ⚠️ デプロイエラーが発生しました

デプロイ（EtTpqboeh）でエラーが発生しています。

## 📋 エラーの確認

### ステップ1: Vercelのログを確認

1. **Vercelダッシュボードでプロジェクトを開く**
   - https://vercel.com/dashboard

2. **「Deployments」タブをクリック**

3. **エラーが発生したデプロイ（EtTpqboeh）をクリック**

4. **「Logs」タブをクリック**

5. **エラーメッセージを確認**
   - ビルドエラーか実行時エラーかを確認
   - エラーメッセージの内容をコピー

## 🔧 よくあるエラーと対処法

### エラー1: ビルドエラー

**症状**: TypeScriptのエラーやビルドのエラー

**対処法**:
1. エラーメッセージを確認
2. ローカルで `npm run build` を実行してエラーを確認
3. エラーを修正して再コミット・プッシュ

### エラー2: 依存関係のインストールエラー

**症状**: `npm install` でエラーが発生

**対処法**:
1. Vercelのプロジェクト設定で「Install Command」を確認
2. 必要に応じて `npm install --legacy-peer-deps` に変更
   - Settings → Build & Development Settings → Install Command

### エラー3: 環境変数のエラー

**症状**: 環境変数が正しく設定されていない

**対処法**:
1. Settings → Environment Variables で環境変数を確認
2. すべての環境変数が設定されているか確認
3. 値が正しいか確認

### エラー4: ビルドコマンドのエラー

**症状**: ビルドコマンドが失敗する

**対処法**:
1. Settings → Build & Development Settings でビルドコマンドを確認
2. デフォルトの `npm run build` が正しいか確認

## 🆘 トラブルシューティング

### エラーログが見つからない場合

1. デプロイをクリックして、詳細を確認
2. 「Logs」タブでエラーメッセージを確認
3. 「Build Logs」と「Runtime Logs」の両方を確認

### ビルドが成功するが実行時エラーが発生する場合

1. 環境変数が正しく設定されているか確認
2. Supabaseの接続を確認
3. ブラウザのコンソール（F12）でエラーを確認

## 🎯 次のステップ

エラーログの内容を確認して、具体的なエラーメッセージを共有してください。それに基づいて、適切な対処方法を案内します。
